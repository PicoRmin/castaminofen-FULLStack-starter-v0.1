# خلاصه پایداری Workspace

این تغییرات با هدف پایدارسازی مخزن قبل از پیاده‌سازی قابلیت RSS انجام شد و فقط روی پیکربندی‌های پایه و سازگاری ساخت متمرکز بود.

## خلاصه مشکلات یافت‌شده
- اسکریپت‌های build بسیاری از بسته‌های workspace با `tsc -p` اجرا می‌شدند و از مرجع‌گذاری TypeScript پشتیبانی نمی‌کردند.
- در این حالت، بسته‌های وابسته در monorepo ممکن بود قبل از آماده‌شدن، توسط اپ‌ها یا بسته‌های دیگر به‌درستی resolve نشوند.
- برای حفظ سازگاری با معماری فعلی، باید از `tsc -b` برای پروژه‌های TypeScript با `references` استفاده می‌شد.

## تغییرات اعمال‌شده
- اسکریپت‌های build در برنامه‌ها و بسته‌های workspace از `tsc -p tsconfig.json` به `tsc -b tsconfig.json` تغییر کرد.
- این کار باعث شد وابستگی‌های workspace به‌صورت ترتیبی و پایدارتر ساخته شوند.
- ساخت کلی repo با `pnpm build` دوباره بررسی و تأیید شد.
- Prisma با `pnpm exec prisma generate` نیز با موفقیت اجرا شد.

## نتیجه
- نصب依ست‌ها با `pnpm install` موفق بود.
- ساخت کل مخزن با `pnpm build` موفق بود.
- تولید Prisma موفق بود.
